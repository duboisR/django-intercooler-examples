Dependent Select

This example has a form with a second drop-down dependent on the value of the first, with the values driven by a server-side data structure.

Explanation

The first dropdown issues a post to /models via a ic-get-to attributes. It does this when the value changes (the default trigger on inputs). It targets the user drop down by CSS ID using the ic-target attribute, and displays a spinner via the ic-indicator attribute.

The server side gets the group that was selected in the input via a request parameter and renders an select with all the user available for that group.
